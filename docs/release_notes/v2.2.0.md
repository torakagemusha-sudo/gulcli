# GUL CLI v2.2.0 Release Notes

Release date: 2026-07-01

## Summary

GUL v2.2.0 closes the execution gap between documented CLI behavior and shipped capability. Both the Python package and native C++ CLI now perform real file-backed `validate` and `infer`, emit schema-conformant JSON envelopes, and support scenario-driven dataset generation with provenance.

## Added

- Executable `validate` and `infer` on Python and native C++ paths
- Versioned JSON schemas under `schemas/` with schema validation tests
- Golden output fixtures and expanded CI (`runtime-ci`, `package-ci`, `cpp-runtime-ci`)
- Python fact environment (`--facts`) for atom predicate evaluation
- Native C++ `FactEnvironment` with `--facts` infer parity
- Scenario-driven dataset generation (`--scenario`, `--stats`) with provenance extensions
- Spec-linked dataset generation (`--spec <path>`) tying samples to GUL policy specs
- Temporal trace enrichment for `ALWAYS`, `EVENTUALLY`, and `UNTIL` (structural approximation)
- Bundled `schema_validate` helper module (installed with the package for CI/tests)

## Changed

- Package version finalized at `2.2.0`
- Dataset generator version aligned to `2.2.0`
- CI test runner consolidated to `python -m unittest discover -s tests`
- `package-ci` build step uses `python -m build` (not `python -m pip build`)

## Fixed

- `schema_validate` import failure when running `python -m unittest tests.test_schemas` after editable install
- Wheel build workflow command syntax in `package-ci.yml`

## Known limitations

- Temporal operators use structural approximation, not full LTL model checking
- Dataset scenarios are template families; `--spec` links provenance and baseline inference for permit-path samples
- Windows C++ CI sanity job runs on `windows-latest` in `cpp-runtime-ci.yml`

## Native Windows binary

Rebuild the checked-in `gul.exe` on Linux (mingw-w64):

```bash
bash cpp/scripts/build-gul-exe.sh
```

Or on Windows:

```bash
cmake -S cpp -B cpp/build -DCMAKE_BUILD_TYPE=Release
cmake --build cpp/build --config Release
```

## Upgrade

```bash
python3 -m pip install -e .
```

Native build (Linux):

```bash
cmake -S cpp -B build -DCMAKE_CXX_COMPILER=g++-12 && cmake --build build
```
